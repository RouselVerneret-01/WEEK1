1.a. listing the steps needed to create a Web form and adding a server control to the form.
                   First, you need to have visual studio install on your computer, window or Apple computer with all requirement to allow your web form to work. After that you will follow those steps below.
  Open Visual Studio 
 Create a New Project 
Click File → New → Project 
Select ASP.NET Web Application (.NET Framework) 
Click Next 
  Name the Project 
Enter a project name 
Choose a location 
Click Create 
 Select Web Forms Template 
In the template selection window, choose Web Forms 
Click Create 
 Add a New Web Form 
In Solution Explorer, right-click the project 
Select Add → Web Form 
Enter a name (e.g., Default.aspx) 
Click Add

2.a. describing the advantages of partitioning an ASP.NET page.
Partitioning an ASP.NET page separates it into logical components (e.g., .aspx UI, code-behind .aspx.cs, user controls, and components). This improves structure, maintainability, and scalability.


2.b. analyzing the process of creating and using code-behind pages.
Code-behind is a model for developing web pages, that physically separates the user interface markup (HTML or XAML) from the underlying application logic and event handling code. This separation allows for cleaner code, easier maintenance, and better collaboration between designers and programmers. The process follow those steps, creating the makeup file, create the code behind file link the files, define controls and logic.

2.c. analyzing the process of creating and using components.
Creating and using components is a systematic process aimed at building modular, reusable, and maintainable systems.
2.d. explaining the need for code-behind pages.
The code behind use to separate the user interface the underlying business logic, with benefit of organize,and maintenance.

3.b. explaining the process of connecting to an ADO data source.
Basically Connecting to an ADO (ActiveX Data Objects) data source involves establishing a connection object, typically ADODB.connection or in .NET and passing a connection string containing provider information, data source location, and authentication credentials. The process is finalized by opening the connection using the .Open()  method, enabling database interaction, and closing it
3.d. describing the process of displaying data on list-bound controls.
The general process for binding data to list-bound controls typically involves, define   Data source, Add  a Data source control, place the list control, configure the blinding, map the data, define layout and appearance.
